**File structure (follow reference_tutorial.py pattern):**

1. **Module docstring** explaining unstructured vs structured pruning, when to use each, and what the tutorial demonstrates.

2. **Imports block:**
   ```python
   import os, sys, logging, tempfile
   sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
   import torch
   import torch.nn as nn
   import torch.nn.utils.prune as prune
   from utils import (setup_logging, benchmark, print_benchmark_table,
                      SimpleCNN, get_sample_batch, get_device, print_device_info)
   ```

3. **Constants:**
   ```python
   SPARSITY_LEVELS = [0.2, 0.5, 0.7, 0.9]
   BATCH_SIZE = 64
   NUM_INFERENCE_ITERATIONS = 100
   FINE_TUNE_EPOCHS = 2
   LEARNING_RATE = 0.001
   ```

4. **Helper functions:**

   - `measure_model_size(model)` -> dict with "param_count" (int) and "file_size_mb" (float). Uses `sum(p.numel() for p in model.parameters())` for param count and `tempfile.NamedTemporaryFile` + `torch.save(model.state_dict(), f.name)` + `os.path.getsize` for file size. Returns both values.

   - `@benchmark` decorated `run_inference(model, inputs, num_iterations)` that runs `model(inputs)` in a loop under `torch.inference_mode()` for NUM_INFERENCE_ITERATIONS iterations. Returns None (timing captured by decorator).

   - `fine_tune(model, device, epochs, lr)` that creates synthetic training data via `get_sample_batch(batch_size=BATCH_SIZE, device=device)`, sets up `nn.CrossEntropyLoss()` and `torch.optim.SGD(model.parameters(), lr=lr)`, and runs `epochs` training iterations. Logs "Fine-tuning for {epochs} epochs..." via logger.

   - `build_pruned_model(original_model, prune_ratio, device)` for structured pruning (per D-01). This function:
     a. Deep-copies the original model
     b. Applies `prune.ln_structured(conv, name='weight', amount=prune_ratio, n=1, dim=0)` to features.0
     c. Identifies surviving output channel indices from the weight mask
     d. Builds a new SimpleCNN-like model with physically smaller layers:
        - New features.0: Conv2d(3, num_surviving, 3, padding=1) with weights sliced from original
        - New features.3: Conv2d(num_surviving, 64, 3, padding=1) with input channels sliced
        - features.6 and classifier layers copied unchanged
     e. Returns the new smaller model on device

5. **`main()` function with these sections:**

   **Section A: Setup** (print section header per convention)
   - `device = get_device()`, `print_device_info()`
   - Create baseline SimpleCNN, move to device, set eval mode
   - Log total parameter count: `logger.info(f"Baseline model: {total_params:,} parameters")`
   - Measure baseline size with `measure_model_size()`
   - Create sample input with `get_sample_batch(batch_size=BATCH_SIZE, device=device)`
   - Measure baseline inference with `run_inference(model, inputs, NUM_INFERENCE_ITERATIONS)`

   **Section B: Unstructured Pruning** (per D-01, D-02, D-03)
   - Print section header: "PART 1: Unstructured Pruning (L1 Magnitude)"
   - Log explanation: "Unstructured pruning zeros out individual weights by magnitude. The weight tensor stays the SAME SIZE -- zeros are stored in a dense tensor. Standard GPU kernels process all elements including zeros, so there is NO inference speedup on standard hardware."
   - For each sparsity in SPARSITY_LEVELS:
     a. Deep-copy baseline model
     b. Apply `prune.l1_unstructured(module, name='weight', amount=sparsity)` to all Conv2d and Linear layers
     c. Fine-tune with `fine_tune(model_copy, device, FINE_TUNE_EPOCHS, LEARNING_RATE)` (per D-02)
     d. Make pruning permanent: iterate all pruned layers and call `prune.remove(module, 'weight')`
     e. Set eval mode
     f. Measure size with `measure_model_size()`
     g. Log: sparsity level, zero count vs total weight count, param count (unchanged), file size
     h. Measure inference with `run_inference()`
     i. Collect results for benchmark table with name like f"Unstructured {int(sparsity*100)}%"
   - **Critical log (per D-03):** After the loop, print: "NOTE: Unstructured pruning does NOT reduce inference time on standard GPUs. The weight tensors remain dense (same shape), and standard CUDA kernels process all elements including zeros. Sparse hardware (e.g., NVIDIA A100 structured sparsity support) is required for actual speedup from unstructured sparsity."

   **Section C: Structured Pruning** (per D-01, D-02)
   - Print section header: "PART 2: Structured Pruning (Channel Removal)"
   - Log explanation: "Structured pruning physically removes entire channels/filters, producing genuinely smaller layers. This results in real inference speedup because the GPU has less computation to perform."
   - For prune_ratio in [0.25, 0.5] (demonstrate 25% and 50% channel removal on features.0):
     a. Call `build_pruned_model(baseline_model, prune_ratio, device)`
     b. Fine-tune the pruned model (per D-02)
     c. Set eval mode
     d. Measure size with `measure_model_size()`
     e. Log: channels removed, new param count, file size
     f. Measure inference with `run_inference()`
     g. Collect results with name like f"Structured {int(prune_ratio*100)}% channels removed"

   **Section D: Benchmark Comparison Table** (per D-09)
   - Build results list including baseline + all unstructured variants + all structured variants
   - Each entry: `{"name": label, "time_seconds": timing, "memory_mb": memory}`
   - Also print a separate model size comparison table using print() with formatted columns showing: Configuration | Params | File Size (MB)
   - Call `print_benchmark_table(results)` for the inference speed table
